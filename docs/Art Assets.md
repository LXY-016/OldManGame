# 美术资源与视觉设计规范 (Art Assets & Visual Design)

> **文档版本**: V1.0
> **目的**: 统一归档所有视觉风格、3D 资源、UI 元素及特效规范。

## 1. 整体视觉风格 (Overall Visual Style)
*   **美术风格**: **精致体素风 (Refined Voxel Art)**。类似于《我在七年之后等你》，保留方块感的同时，追求物体的高辨识度与精致感。
*   **视角**: 固定 2.5D 等轴侧视角 (45度角)。
*   **建模特征**: 
    *   **高细节体素 (High-Res Voxel)**: 虽由方块构成，但方块精度较高，确保物体结构清晰可见（例如：能看清制服的领口、轮椅的轮辐）。
    *   **材质**: 还是以纯色为主，但在关键部位（如脸部表情、衣服花纹）增加体素细节，**拒绝抽象化**。
*   **氛围**: 干净、精致、明亮。
*   **色彩**: 保持温和的马卡龙色系，但通过更丰富的光影层次来提升精致度。
*   **光影**: 启用更高级的渲染效果（AO环境光遮蔽、软阴影），消除“廉价感”。

## 2. 场景美术资源 (Environment Assets)

### 2.1. 主界面背景 (Start Screen / Layer 0)
*   **概念**: 展现社区全景的温馨画面。
    *   风格化的居民楼。
    *   向玩家打招呼的社工角色。
    *   柔和的光照/泛光效果 (Bloom)。
    *   **UI层**: 
        *   **标题**: 大尺寸艺术字。
        *   **主按钮 (Start)**: 亮色高亮，比退出按钮更醒目。
        *   **次要按钮 (Exit)**: 较小或低饱和度颜色。

### 2.2. 游戏关卡场景 (Game Level / Layer 1)
*   **设定**: 老旧小区 (Chinese Residential Community)。
*   **核心资产**:
    *   **地面**: 
        *   **网格化路径 (Grid Path)**: 道路转弯处呈直角或阶梯状，但**边缘倒角处理**，不割手。
        *   **地块 (Tiles)**: 清晰的地砖纹理，有厚度感。
    *   **建筑**: 
        *   **主要背景**: 像**精致乐高模型**一样的居民楼。每扇窗户都有窗框细节，阳台上有晾衣架等生活气息的体素装饰。
    *   **道具**:
        *   **植被**: 体素树木层级丰富，**不只是一团绿**，要有树叶的疏密变化。
        *   **长椅/设施**: 结构完整，能看清扶手和靠背的细节。
    *   **生成点 (Spawner)**: 具有景深的楼栋门洞，内部有黑色渐变。

### 2.3. 关卡选择界面 (Level Select / Layer 2)
*   **概念选项**:
    *   **社区长卷地图**: 横向展开的社区街道全景 (Horizontal Scroll)。
    *   **节点设计**: 每一个关卡是一个具体的“单元楼门洞”或“社区服务点”。
*   **状态表现**:
    *   **未解锁 (Locked)**: 灰色，被迷雾或锁链遮挡。
    *   **已解锁 (Unlocked)**: 亮色，呼吸动画。
    *   **已通关 (Cleared)**: 显示“通关标记”或“最高服务人数”。

## 3. 角色美术资源 (Character Assets)

### 3.1. 社工小队 (Social Workers - Player Units)
### 3.1. 社工小队 (Social Workers - Player Units)
**核心机制**: 社工平时为 **UI 2D 图标**，拖拽释放后转化为 **3D 体素模型**。

#### A. UI 资源需求 (UI Assets)
主要用于底部操作栏与拖拽过程。
*   **[Icon_Normal] 待机状态**:
    *   **描述**: 彩色、高亮、背景清晰的社工大头贴或半身像。
    *   **风格**: 精致像素画或体素模型的渲染图。需清晰展示职业特征（如强壮者的安全帽、医生的眼镜）。
*   **[Icon_Dragging] 拖拽状态**:
    *   **描述**: 跟随鼠标移动的图标。
    *   **处理**: 建议使用 `Icon_Normal` 的半透明版本 (Alpha 70%) 或 略微放大的版本。
*   **[Icon_Cooldown] 冷却状态**:
    *   **描述**: `Icon_Normal` 的去色/灰度版本。
    *   **覆盖层**: 需要一个扇形或方形的动态遮罩 (Mask) 来表现剩余时间。

#### B. 3D 资源需求 (3D Scene Assets)
仅在任务执行阶段 (Working Phase) 出现。
*   **[Model_Voxel] 体素模型**:
    *   **风格**: **高精度体素人 (Detailed Voxel Humans)**。
    *   **细节要求**:
        *   **头部**: 必须有简单的像素表情（眨眼、说话嘴型）。
        *   **身体**: 肢体分明，手部有“手掌”形状，膝盖有关节暗示。
        *   **装备**: 必须模型化（如背在背上的大扳手、挂在脖子上的听诊器），不能只是贴图。
*   **[Animations] 关键动画**:
    *   `Spawn` (登场): 烟雾中变出 / 快速从屏幕外跑入。
    *   `Work` (工作): 针对不同属性的特定动作（举重、记录、维修）。
    *   `Exit` (退场): 敬礼/挥手后消失 / 跑出屏幕。

#### C. 角色区分 (Character Variations)
*   **Worker-A (强壮/STR)**: 🟤 棕色系。特征：安全帽、工装裤、大扳手/箱子。
*   **Worker-B (健谈/CHA)**: 🟢 绿色系。特征：便服、志愿者马甲、记录板。
*   **Worker-C (聪明/INT)**: 🔵 蓝色系。特征：眼镜、白大褂/急救包、平板电脑。

### 3.2. 独居老人 (Elders - NPCs)
*   **风格**: **体素方块老人**。
*   **特征**: 通过白色方块头发、胡子或者佝偻的方块身体来区分。
*   **变体**:
    *   拄拐杖行走。
    *   坐轮椅。
    *   提着菜篮子。
*   **动画**:
    *   `Walk` (行走): 缓慢移动。
    *   `Idle/Waiting` (等待): 站立并显示需求气泡。
    *   `Happy/Thanks` (感谢): 任务成功后挥手或鞠躬。
    *   `Sad/Leave` (离开): 任务失败后摇着头离开。

## 4. UI 界面与图标 (UI & Icons)

### 4.1. 需求气泡 (Task Bubbles)
使用 **世界空间 UI (World Space UI)**，采用 **图文组合 (Icon + Text)** 的形式以提升信息传达效率与美观度。

*   **布局设计 (Visual Layout)**:
    *   **容器**: 带有圆角或像素切角的长方形气泡面板，底部有指向老人的小三角 (Anchor)。
    *   **组合方式**: **左图右文**结构。左侧为大尺寸像素图标，右侧为精简的**事件名称**。
    *   **字体**: 使用清晰粗体的像素字体 (Pixel Font)，颜色为高对比度的白色或深色。
    *   **动态美感**:
        *   **呼吸浮动**: 气泡整体有轻微的上下正弦波浮动。
        *   **入场动画**: 气泡弹出时有弹性缩放 (Pop-in animation)。
        *   **紧急状态**: 当倒计时 < 30% 时，气泡背景开始在原色与红色之间脉冲闪烁，并伴随轻微抖动。

*   **样式定义**:
    *   **E-01 (起居)**: 🟤 暖木色板。图标：[箱子] + 文字：**搬运**。
    *   **E-02 (社交)**: 🟢 新鲜草绿。图标：[气泡] + 文字：**聊天**。
    *   **E-03 (维修)**: 🔵 工业深蓝。图标：[扳手] + 文字：**维修**。
    *   **E-04 (医疗)**: 🟢 治愈浅绿。图标：[十字] + 文字：**陪诊**。
    *   **E-05 (紧急)**: 🔴 警示暗红 + **流光边框**。图标：[感叹号] + 文字：**急救!**。

### 4.2. HUD 元素
*   **倒计时**: 像素字体 (Pixel Font) 显示。

### 4.3. 交互反馈 (Feedback VFX)
强调操作的“爽快感”与信息的“准确性”。

*   **选中/悬停 (Hover/Select)**:
    *   **底部头像**: 鼠标悬停时上浮 (Y轴+5px) 并高亮。
    *   **老人目标**: 鼠标悬停时，脚下出现 **白色方块光标 (White Square Cursor)**，并跟随呼吸节奏缩放。
*   **拖拽交互 (Drag & Drop)**:
    *   **匹配高亮 (Attribute Match)**: 当拖拽社工时，场上需求完美匹配的老人，**气泡放大 1.2倍** 并且显示 **求助 (Help!)** 动效。
    *   **有效目标悬停**: 
        - **普通目标**: 金色描边 (Golden Rim)。
        - **完美匹配**: **亮绿色描边 (Green Rim)** + 强烈的粒子吸引特效。
    *   **无效释放**: 拖拽物播放快速缩小的“噗”消失动画，或弹回底部栏的轨迹动画。
*   **任务结算 (Result)**:
    *   **成功 (Success)**:
        *   **触发条件**: 社工服务进度条 (T-Action) **读满 (100%)**。
        *   **视觉**: 老人头顶弹出像素风格的 **`+1`** 浮动文字 (金色)。周围迸发少量 **方形彩屑粒子 (Confetti)**。
        *   **动作**: 社工与老人同时播放“开心/感谢”动画。
    *   **失败 (Fail)**:
        *   **触发条件**: 老人等待倒计时 (T-Request) **归零 (0s)** 且未获得服务。
        *   **视觉**: 需求气泡破碎，老人头顶出现 **灰色乱线团** (表达心情糟糕) 或 紫色骷髅图标。
        *   **动作**: 老人摇头叹气并离场。
*   **冷却与错误 (Cooldown & Error)**:
    *   **冷却遮罩**: 底部头像上覆盖深色半透明扇形扫描 (Radial Fill)。
    *   **非法操作**: 点击冷却头像或将社工拖到无效区域时，头像左右轻微晃动 (Shake)，并弹出“休息中/无效”的轻量提示文本。

## 5. 动画与特效 (Animation & VFX)

### 5.0. 通用 UI 交互 (General UI Interaction)
*   **设计语言**: **拟物化像素 UI (Tactile Pixel UI)**。强调按钮的“厚度”与点击的“段落感”。
*   **按钮 (Buttons)**:
    *   **构造**: 纯色主块 + 深色体素边框 (底部厚度 4px)，营造物理按键感。
    *   **交互态**:
        *   **悬停 (Hover)**: 按钮整体上浮 (Y-2px)，底部阴影拉长，文字发光。
        *   **按下 (Pressed)**: 按钮下沉 (Y+2px)，阴影被压缩消失，伴随清脆的 **机械键盘音效**。
*   **转场 (Transitions)**:
    *   **场景切换**: 使用 **马赛克像素化过场 (Pixelate Fade)**，画面由清晰变马赛克再变清晰，而非简单的黑屏。

### 5.1. 结算反馈 (Settlement Visuals)
*   **胜利 (Win)**:
    *   **核心氛围**: **暖阳高照 (Warm & Glorious)**。
    *   **视觉表现**:
        *   结算面板背后有旋转的 **像素上帝光 (Pixel God Rays)**。
        *   屏幕上方撒下有物理碰撞的 **体素彩带 (Voxel Confetti)**，会落在 UI 框体上弹跳。
    *   **角色**: 所有社工在背景中排成一排，播放 **波浪式跳跃 (Wave Jump)** 庆祝动画。
*   **失败 (Lose)**:
    *   **核心氛围**: **冷雨凄凄 (Cold & Gloomy)**。
    *   **视觉表现**:
        *   画面整体 **去色 (Desaturate)** 并叠加蓝紫色滤镜。
        *   前景下起淅沥沥的 **蓝色体素雨 (Voxel Rain)**。
    *   **角色**: 画面定格在老人失望离开的 **孤单背影** 上。
