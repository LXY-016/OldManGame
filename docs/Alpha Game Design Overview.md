# OldManGame 游戏玩法概述

## 1. 游戏背景与立意
**主题**：关爱独居老人，传递社区温暖。
**角色**：玩家扮演一支专业的“暖阳社工小队”调度员。
**场景**：2.5D 俯视角 (Isometric) 下的温馨老旧小区，居住着性格各异的独居长者。

## 2. 核心玩法 (Core Loop)
游戏的每一关（一天）都遵循以下循环：
1.  **观察 (Observe)**：
    *   小区中的老人头顶随机弹出**需求气泡**（如：购买药品、家电维修、心理疏导、烹饪膳食）。
    *   气泡颜色代表紧急程度，随时间流逝变红（代表老人心情焦躁即便将降低健康值）。

2.  **决策与执行 (Dispatch)**：
    *   玩家从屏幕下方的**社工列表**中，**拖拽**合适的社工角色。
    *   将社工放置到有需求的老人身边。
    *   **匹配机制**：每个社工拥有一个**主属性**（如：强壮、健谈、聪明）。若属性与老人的需求类型（如：搬运需强壮）完美匹配，将大幅减少解决时间；否则耗时较长。

3.  **结算与反馈 (Feedback)**：
    *   需求解决，气泡消失，**服务人数 +1**。
    *   若超时未解决，老人生气离开，不计入服务人数。

## 3. 关卡与挑战体系
*   **限时挑战**：采用关卡制，MVP版本共 **10个关卡**。每关限时（如 120秒）。时间结束时结算服务人数。
*   **难度动态递进 (Flow)**：
    *   **初期 (Lv 1-3)**：需求单一，刷新频率较低，旨在教学与上手。
    *   **中期 (Lv 4-7)**：多重需求并发，刷新频率加快。需考验玩家的快速分配与人员调度。
    *   **后期 (Lv 8-10)**：**困难事件概率 (hardEventProb)** 大幅提升，刷新频率极高。考验在极限压力下的优先级判断与取舍。

## 4. 评价指标 (Evaluation)
本游戏摒弃复杂的积分规则，采用最朴素的评价标准：**服务人数**。
*   **服务计数**：每成功解决一个老人的需求，计数 +1。
*   **关卡目标**：每关设有最低服务人数要求（例如：需服务 10 位老人才能通关）。
*   **失败条件**：倒计时结束时，若服务人数未达标，则判定关卡失败。

## 5. 视觉与操作体验
*   **操作**：纯鼠标操作。核心交互为**“点击-拖拽-释放” **。
*   **视角**：固定 2.5D 等轴侧视角，展现精致可爱的微缩小区模型。
*   **UI 布局**：
    *   顶部：倒计时、服务人数。
    *   中部：游戏场景（小区）。
    *   底部：社工小队待命栏（可拖拽区域）。
