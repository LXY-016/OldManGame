# OldManGame 游戏玩法概述

## 1. 游戏背景与立意
**主题**：关爱独居老人，传递社区温暖。
**角色**：玩家扮演一支专业的“暖阳社工小队”调度员。
**场景**：2.5D 俯视角 (Isometric) 下的温馨老旧小区，居住着性格各异的独居长者。

## 2. 核心玩法 (Core Loop)
游戏的每一关（一天）都遵循以下循环：
1.  **观察 (Observe)**：
    *   小区中的老人头顶随机弹出**需求气泡**（如：购买药品、家电维修、心理疏导、烹饪膳食）。
    *   气泡颜色代表紧急程度，随时间流逝变红（代表老人心情焦躁即便将降低健康值）。

2.  **决策与执行 (Dispatch)**：
    *   玩家从屏幕下方的**社工列表**中，**拖拽**合适的社工角色。
    *   将社工放置到有需求的老人身边。
    *   将社工放置到有需求的老人身边。
    *   **匹配机制**：每个社工拥有一个**主属性**（如：强壮、健谈、聪明）。若属性与老人的需求类型（如：搬运需强壮）完美匹配，将大幅减少解决时间；否则耗时较长。

3.  **结算与反馈 (Feedback)**：
    *   需求解决，气泡消失，获得积分与“老人满意度”。
    *   若超时未解决，老人健康值下降，扣分。

## 3. 关卡与挑战体系
*   **限时挑战**：采用关卡制，每关限时（如 120秒）。时间结束时结算当日总分。
*   **难度动态递进 (Flow)**：
    *   **初期**：需求单一，刷新频率低，旨在教学。
    *   **中期**：多重需求并发，需要考验玩家的手速与人员调度策略。
    *   **后期**：加入突发事件（如流感来袭需集体测温），不仅考验速度，更考验优先级判断。

## 4. 评分与激励系统
为了体现“餐厅经营类”游戏的爽快感，设计深度的评分规则：
*   **基础分**：完成单一任务。
*   **COMBO 连击**：短时间内连续完成多个任务，触发 `x2`, `x4` 分数倍率。
*   **完美响应 (Perfect)**：在需求气泡弹出的 3秒内做出响应。
*   **关卡评级**：C (及格) / B (良好) / A (优秀) / S (完美)。
    *   *只有达到“C级”以上才能解锁下一关卡。*
*   **全球排行榜**：联网上传关卡最高分，与全服社工比拼服务质量。

## 5. 视觉与操作体验
*   **操作**：纯鼠标操作。核心交互为**“点击-拖拽-释放” (Drag & Drop)**。
*   **视角**：固定 2.5D 等轴侧视角，展现精致可爱的微缩小区模型。
*   **UI 布局**：
    *   顶部：倒计时、当前分数、连击数。
    *   中部：游戏场景（小区）。
    *   底部：社工小队待命栏（可拖拽区域）。
